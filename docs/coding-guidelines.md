# Coding Guidelines for TODO App

This document summarizes the coding style and quality principles for the TODO application project.

## Coding Style & Quality Principles

1. **General Formatting Rules**
   - Use consistent indentation (spaces or tabs as specified in project settings).
   - Follow standard naming conventions for variables, functions, and classes.
   - Keep lines concise; avoid overly long lines and complex expressions.

2. **Import Organization**
   - Group imports by origin: external libraries first, then internal modules.
   - Remove unused imports and keep import statements tidy.
   - Use absolute imports where possible for clarity.

3. **Linter Usage**
   - Use a linter (such as ESLint) to enforce code quality and style rules.
   - Address all linter warnings and errors before committing code.
   - Configure linter rules to match project standards and review regularly.

4. **Best Practices**
   - Apply the DRY (Don't Repeat Yourself) principle: avoid code duplication by reusing functions and components.
   - Write clear, maintainable code with descriptive comments where needed.
   - Refactor code regularly to improve readability and efficiency.
   - Prioritize simplicity and clarity over cleverness.
